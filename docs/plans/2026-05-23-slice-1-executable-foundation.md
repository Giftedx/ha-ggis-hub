# Slice 1 Executable Foundation Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Convert the documentation-only foundation into a real, reproducible Rust/WASM + TypeScript/Vite project skeleton without adding gameplay scope or renderer dependency.

**Architecture:** The Rust workspace owns deterministic core crates. `hub-core` is browser-independent and starts with a tiny tested public identity API so the workspace has a real compile/test surface. `hub-wasm` owns the browser/WASM boundary and depends on `hub-core`. The TypeScript/Vite host is intentionally framework-free and renderer-free in this slice; it proves strict build wiring and a clean DOM shell only.

**Tech Stack:** Rust 2024 edition, wasm-bindgen, TypeScript strict mode, Vite, pnpm. No renderer, UI framework, router library, CSS framework, game engine, or convenience dependency in Slice 1.

---

## Non-negotiables

- Do not add gameplay logic in this slice.
- Do not pick Canvas2D vs PixiJS in this slice.
- Do not add React, Svelte, Vue, Phaser, Tailwind, router libraries, input libraries, or state-management packages.
- Do not claim a planned gate is current until it runs.
- Every created source file must either be exercised by a build/test gate now or be explicitly documented as a placeholder-free structural file.
- Keep all line endings LF through `.gitattributes`.

## Acceptance checklist

- Git repository exists on branch `main`.
- Baseline documentation commit exists before implementation scaffold.
- `.gitattributes` and `.gitignore` protect repository hygiene.
- Baseline documentation commit exists and is visible in `git log --oneline -1` before scaffold changes.
- Rust workspace builds and tests.
- `Cargo.lock` is generated and committed because this is an application repository.
- `hub-core` has a small tested API.
- `hub-wasm` compiles for native check/test and for `wasm32-unknown-unknown` check.
- TypeScript strict mode passes.
- `pnpm-lock.yaml` is generated and committed.
- Vite production build passes.
- Markdown local links pass.
- Docs accurately say the executable skeleton now exists, while still marking renderer/gameplay/deployment gates as planned.

## Task 2: Rust workspace skeleton

**Objective:** Create a minimal Rust workspace with `hub-core` and `hub-wasm` crates.

**Files:**
- Create: `Cargo.toml`
- Create: `Cargo.lock` (generated by Cargo, committed)
- Create: `crates/hub-core/Cargo.toml`
- Create: `crates/hub-core/src/lib.rs`
- Create: `crates/hub-wasm/Cargo.toml`
- Create: `crates/hub-wasm/src/lib.rs`

**Implementation requirements:**
- Workspace resolver must be `2`.
- Rust edition must be `2024`.
- `hub-core` must have no runtime dependencies.
- `hub-core` must expose:
  - `PROJECT_NAME: &str`
  - `CORE_API_VERSION: u32`
  - `core_identity() -> CoreIdentity`
  - `CoreIdentity { project_name: &'static str, api_version: u32 }`
- `hub-core` must include tests for identity values.
- `hub-wasm` must depend on `hub-core` and `wasm-bindgen` only.
- `hub-wasm` must set `[lib] crate-type = ["cdylib", "rlib"]` so it can produce WASM while remaining checkable/testable by Cargo.
- `hub-wasm` must expose a tiny `hub_core_api_version() -> u32` wasm binding.

**TDD process:**
1. Write the `hub-core` identity tests first and run them to observe the expected failure before implementation.
2. Implement the minimal `hub-core` API.
3. Run the narrow test, then the full workspace gates.

**Verification:**
- `cargo fmt --all -- --check`
- `cargo test --workspace`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `RUSTFLAGS="-D warnings" cargo check --workspace --target wasm32-unknown-unknown`

- CSS must be handwritten.
- No renderer dependency.

**Verification:**

## Task 5: Final verification and commit

**Objective:** Prove the slice and commit it as a coherent batch.

**Verification commands:**

```bash
cargo fmt --all -- --check
cargo test --workspace
cargo clippy --workspace --all-targets -- -D warnings
RUSTFLAGS="-D warnings" cargo check --workspace --target wasm32-unknown-unknown
pnpm install --frozen-lockfile
pnpm exec tsc --noEmit
pnpm exec vitest run
pnpm run build
python - <<'PY'
from pathlib import Path
import re
errors=[]
for md in Path('.').rglob('*.md'):
    if '.git' in md.parts or 'node_modules' in md.parts:
        continue
    text=md.read_text(encoding='utf-8')
    for m in re.finditer(r'\[[^\]]+\]\(([^)]+)\)', text):
        url=m.group(1).strip()
        if '://' in url or url.startswith('#') or url.startswith('mailto:'):
            continue
        target=url.split('#',1)[0]
        if target and not (md.parent/target).resolve().exists():
            errors.append(f'{md}: {url}')
if errors:
    raise SystemExit('\n'.join(errors))
print('links_ok')
PY
```

**Commit:**

```bash
git add -A
git commit -m "chore: add executable foundation skeleton"
```
